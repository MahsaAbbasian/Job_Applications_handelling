Project Overview Document

Project Name

PlanCraft Kids

Project Description

PlanCraft Kids is a smart chores app designed by parents for their kids. It includes a gamified reward system to encourage children to complete tasks.

Development Status

The project currently has a login and registration system in progress, utilizing Firebase for authentication. There is also work underway on a "Parent Dashboard" which will be further developed once the authentication flow is complete.

Firebase Setup Status

  • The google-services.json is configured with the project, indicating Firebase has been connected to the project.
  • The Firebase project number is 475003133419, and the storage bucket is plan-craft-kids.appspot.com.

Code Branches

  • code_review_error: This branch contains the latest pushed code that may have unresolved errors needing review.
  • development2_parent_dashboard_wip: Work in Progress (WIP) branch for developing the parent dashboard feature after the login page and registration are complete.

Authentication Implementation

  • Sign up and login functionality with options to sign in with Google and Facebook.
  • Firebase Authentication is set up to handle the user management.

User Interface

  • A login page with buttons for "Sign Up" and "Log In".
  • A sign-up page for new users to create an account with email/password or with Google/FB.
  • A login page for existing users with email/password fields and Google/FB options.
  • Error handling UI, indicating when login fails (e.g., "Password error").

Immediate Next Steps

  1. Review and resolve any errors in the code_review_error branch, particularly related to Firebase Authentication.
  2. Ensure the user interface reflects the designs provided, matching the layout, color scheme, and functionality.
  3. After resolving errors, merge the changes into the development2_parent_dashboard_wip branch to continue with the parent dashboard development.

How to Continue

  1. Clone the repository from GitHub.
  2. Switch to the code_review_error branch using Git commands.
  3. Review the MainActivity.kt, Color.kt, Theme.kt, and Type.kt files for errors, particularly related to Firebase Authentication.
  4. Use the provided designs to guide UI development and ensure a consistent user experience.
